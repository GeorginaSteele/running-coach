# Action: Plan the next training block

Use this action when the current block is ending (typically Week 4 of a 4-week block) or when the athlete asks "what's the next block look like" / "we need to plan after Oxford" / "let's structure the next phase."

A block is typically 4 weeks: 3 build + 1 cutback. Multi-block builds toward race day.

## Steps

1. **Review the closing block.** Read all journal entries from the block. What worked, what didn't, what's the athlete's current state? Were the prescribed paces hit? Did HR caps get respected? Any injury near-misses?

2. **Reassess pace zones.** If a parkrun PR happened during the block, or aerobic HR has dropped at a given pace, update the zones. Use the most recent race effort to recalibrate:
   - 5K time × ~2.18 → predicted half time (with appropriate caveats)
   - HM pace = ~95% of threshold pace = ~90% of 5K pace
   - Easy zone = 75–80% of HM pace, capped by HR ≤Zone 2

3. **Check race timeline.** How many weeks until the goal race? What's the appropriate stage:
   - **Base phase:** if >12 weeks out — focus on aerobic volume and consistency.
   - **Build phase:** 8–12 weeks out — introduce structured tempo and HM-pace work.
   - **Specific phase:** 4–8 weeks out — race-pace specificity, longest long runs, fitness check races.
   - **Taper:** last 2 weeks — volume drops, intensity stays.

4. **Hold the volume cap.** If the athlete has a self-imposed weekly km cap, respect it across all blocks. Don't sneak it up "just for a few weeks."

5. **Draft the block.** Each week needs:
   - Friday quality run (easy / tempo / intervals depending on phase)
   - Saturday parkrun (alternating race / steady)
   - Sunday long run (progressively longer or more specific)
   - Gym + pilates + ballet as already locked
   - Cutback in Week 4

6. **Write to a new markdown file** in the workspace folder, parallel structure to the current block file. Use a clear changelog at the bottom that references decisions made in previous blocks.

7. **Present the block** to the athlete before locking it in. Get explicit sign-off on:
   - Weekly structure
   - Key session targets (paces, distances)
   - Any new elements introduced (e.g. first VO2 session, longest long run)
   - The race-prep landmarks (tune-up race, longest run date, taper start)

## What changes block-to-block

Block 1 (base/return): low quality, mostly easy aerobic, intro strength.
Block 2 (build): introduce tempo at HM pace. Long runs grow.
Block 3 (specific): HM-pace blocks inside long runs, "fast finish" workouts, tune-up parkrun targeting a fitness checkpoint time.
Block 4 (peak + taper): one peak long run (~21km with race-pace work), then volume drop, intensity hold, race.

Adapt the count to the athlete's available training window. For a 21-week plan, 5 blocks of 4 weeks + 1 race week works cleanly.

## Things to explicitly confirm with the athlete

- **Race day pacing strategy.** Even split? Negative split? Cautious-then-empty-tank? This shapes the last 4 weeks of training.
- **Tune-up race?** A 10K or HM-pace race 3–4 weeks before the goal race is a useful fitness checkpoint. Optional but valuable.
- **Travel / life events** that conflict with key sessions.
- **Equipment / venue access** for any new training elements (track, treadmill, hills).

## Pace-zone update example

Athlete's parkrun improved from 22:16 → 21:30 across Block 1. New zones:

| Zone | Old | New |
|---|---|---|
| Easy | 5:30–5:50, HR ≤150 | 5:25–5:45, HR ≤150 |
| HM pace | 4:40–4:50 | 4:35–4:45 |
| Tempo | 4:20–4:35 | 4:15–4:30 |
| 5K | 4:20–4:30 | 4:15–4:22 |

Update via journal entry + plan file. Don't make zones look like they shifted faster than the data actually supports.
