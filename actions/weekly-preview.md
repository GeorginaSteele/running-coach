# Action: Weekly preview

Use this action on Sunday evenings, or whenever the athlete asks "what's the plan this week" / "where are we" / "give me a recap."

## Steps

1. **Pull the last 7 days from Garmin** (`mcp__garmin__get_activities`, limit 20). Include runs, gym, pilates, ballet, walks. Total volume matters.

2. **Read the last 10 journal entries** in `training-journal.md`. The journal is where patterns live — recent observations should shape what you say about next week.

3. **Read the active training plan** (the markdown file for the current block) to ground the preview in the structural plan.

4. **List next 7 days of calendar events.** What's actually scheduled vs. what the plan says should be scheduled? Flag any mismatches.

5. **Write the preview.** Structure below.

6. **Ask one open question at the end** about constraints for the upcoming week — travel, work load, sleep, niggles.

7. **Wait for athlete response.** Adjust calendar based on what she says. Confirm each change before making it.

8. **If structural plan changes are made** (not just per-event tweaks), update the active training plan file with a dated note at the bottom.

## Preview structure

```
## Week of [date range] (Block N / Week M)

**Last week — key takeaways:**
- [2–4 short bullets only. Headlines, not a catalogue. See "What to include in the recap" below.]

**Next week:**
- [Day]: [Session] — [one-line purpose]
- [Day]: [Session] — [one-line purpose]
- [etc.]
- Total: Xkm.

**Things to watch:** [if there's a specific focus for the week — e.g., HR cap test on easy runs, race-effort parkrun, first long run with HM-pace block]

Any constraints I should know about this week — travel, work load, sleep, niggles?
```

## What to include in the recap (and what to leave out)

The recap is *not* a session-by-session log of the week. She already lived through it. Her journal already has the per-session detail. The recap surfaces what matters at a week scale.

**Include only the headlines:**

- **Patterns** (good or bad). "Three easy runs in a row at HR 165+ — this is now the dominant easy-run signature, not a one-off."
- **Notable performances** (PRs, missed targets by significant margins). One line on what it represents in plan terms.
- **Missed sessions**. Don't bury them. "Friday didn't happen — Saturday parkrun substituted as quality."
- **Recovery red flags**. Sub-optimal sleep, climbing HR on easy runs, journaled niggles. Name them.

**Leave out:**

- A line for every session. If Tuesday gym was unremarkable, don't mention it. If Friday went exactly to plan, you can roll it into "everything else went to plan" rather than spelling it out.
- Restating data she's already seen in this week's session reviews. The preview is forward-looking; recap is just for context.

Aim for ~3–5 bullets in the recap section, not 7+. A boring week is allowed to have a one-line recap.

## What to highlight in next week's preview

- **One sentence per session.** Not a re-statement of the calendar event. The *purpose*. Why is this session in the plan?
- **The key session of the week** — the one that, if she only nailed one, would move fitness most.
- **Any planned adjustments coming from last week's data.**

## Tone

Sunday evening tone is reflective + forward-looking. Acknowledge the week, set the stage for the next one. Don't dwell on errors — name them, move on.

If there's nothing dramatic to flag (week went to plan, athlete is healthy, next week is straightforward), keep it short. The preview is information, not theatre.
